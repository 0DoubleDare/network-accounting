# network-accounting
веб-приложение для учёта сетевой инфраструктуры 

## Важное
### Правила работы с ветками
НЕ пушьте свои изменения в главную ветку main! Для этого сущетсвует ветка develop

> [!TIP]
> 
> **Помечайте свои ветки.** 
> Если исправляйте баг, пишите название ветки **bugfix/example**
>
> Для документации - **docs/example**
> 
> Для новых функций **features/example**
>
> Вместо example пишите конкретное назначение вашей ветки, к примеру **docs/update_readme** 
> или **bugfix/fix_db_connection**
> 
> Это нужно для удобства пользования репозиторием в команде

## Правила верстки
В каждом файле view`шки должен быть подключен header.php и footer.php в начале и в конце страницы соотвественно

## Переменные
### Работа с сессиями
id пользователя должен храниться в $_SESSION['id']

роль пользователя в $_SESSION['role']

логин пользователя в $_SESSION['login']
