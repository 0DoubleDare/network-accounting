# Network Accounting

**Network Accounting** — это цифровая экосистема, предназначенная для онтроля над физической инфраструктурой связи. Это проект, который пользоволяет вести контроль над кабинетом и отчёты

---

## Быстрый запуск (Docker)

Проект полностью готов к запуску через Docker Compose что позволяет развернуть его на сервера.

1.  **Соберите и запустите контейнеры:**
    ```bash
    docker-compose up -d --build
    ```
2.  **Доступ к приложению:**
    *   **Веб-интерфейс:** [http://localhost:8080](http://localhost:8080)
    *   **База данных (внешний порт):** `localhost:3307`

> *База данных инициализируется автоматически при первом запуске из файла `install/network_accounting_db.sql`.*

---

## Структура проекта

*   `/app` — основная логика приложения.
    *   `/includes` — подключаемые компоненты (header, footer, auth, functions).
    *   `/view` — визуальные компоненты (верстка)
*   `/config` — конфигурационные файлы (подключение к БД).
*   `/install` — SQL-дампы для развертывания БД.
*   `/storage` — директории для хранения загружаемых файлов (изображения дефектов и точек).
*   `/assets` — статические файлы (CSS, JS, изображения).

## Инструкция по ручной установке (без Docker)

Если вы планируете запускать проект на локальном сервере (например, OpenServer, XAMPP, Apache):

1.  **Требования:** PHP 7.4+ (с PDO MySQL), MySQL / MariaDB.
2.  **Настройка БД:** Создайте БД `network_accounting_db` и импортируйте `./install/network_accounting_db.sql`.
3.  **Конфигурация:** Отредактируйте `config/db.php`:
    ```php
    $db_host = 'localhost';
    $db_name = 'network_accounting_db';
    $user = 'ваш_пользователь';
    $password = 'ваш_пароль';
    ```
4.  **Права доступа:** Убедитесь, что папка `storage/` доступна для записи веб-серверу.

## Концепция и Стек

### 1. Для чего это нужно?
Это специализированная **ERP-система** для телеком-компаний.
*   **Цифровые двойники:** Хранение данных о каждом узле.
*   **Исключение потерь:** Контроль кабеля на складе и фиксация поломок.
*   **Порядок:** Ответы на вопросы «Где?», «В каком состоянии?» и «Кто установил?».

### 2. Стек технологий
*   **Backend:** PHP 7.4/8.x (Native) — высокая скорость и легкость.
*   **Database:** MySQL 8.0 (PDO) — защита от SQL-инъекций.
*   **Infrastructure:** Docker — изоляция среды.
*   **Frontend:** Bootstrap 5, JS — адаптивность под смартфоны инженеров.

## Роли пользователей

1.  **Администратор (Admin):** Архитектор. Доступ к логам, управлению пользователями и полному редактированию.
2.  **Оператор (Operator):** Линейный инженер. Создание точек, фиксация дефектов, списание материалов.

---

## Основные функции

### Учет дефектов
При создании записи фото переименовывается в уникальный хеш и сохраняется в `/storage/defects/`, а путь пишется в БД. Это обеспечивает доказательную базу инцидента.

### Расход материалов
При списании происходит **атомарная операция**:
1. Проверка остатка в `materials`.
2. Уменьшение значения в `materials`.
3. Создание записи об использовании материала в `material_usage`.

### Логирование
Любое изменение вызывает функцию `writeLog()`, которая фиксирует: ID пользователя и описание действия.

## Порядок работы

1.  **Старт:** Разверните систему через Docker.
2.  **Авторизация:** Войдите как обычный пользователь.
3.  **Инфраструктура:** Вкладка "Точки" -> Добавить первый узел.
4.  **Склад:** Вкладка "Материалы" -> Наполните склад остатками.
5.  **Эксплуатация:** 
    *   При поломке: Клик по точке -> "Добавить дефект".
    *   При ремонте: Клик по точке -> "Списание материала".
6.  **Контроль:** Просмотр истории действий в разделе "Логи".
